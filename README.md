# Quizzard Bot 🧠✨ `v0.2.0`

**Quizzard** — это интеллектуальный Telegram-бот для проведения викторин на абсолютно любые темы. Бот динамически генерирует вопросы, варианты ответов и развернутые объяснения к ним с помощью современной языковой модели (LLM) через API.

## 🚀 Основные возможности

- 🎭 **Динамическая генерация викторин:** Пользователь может указать любую интересующую его тему (например, *«История Древнего Рима»*, *«Язык программирования Python»*, *«Кинематограф Кристофера Нолана»*), и искусственный интеллект мгновенно сгенерирует для него тест.
- 🎮 **Три уровня сложности:** Перед началом викторины можно выбрать уровень сложности — `🟢 Легкая`, `🟡 Средняя` или `🔴 Сложная`.
- ⚡️ **Потоковый (Streaming) вывод генерации:** Процесс создания вопросов отображается в чате в реальном времени в виде интерактивного списка.
- 💡 **Объяснения к ответам:** После каждого ответа бот показывает результаты с наглядной цветовой индикацией ответов и выводит подробное объяснение, почему именно этот вариант верен.
- 📊 **Личная статистика:** Пользователи могут отслеживать свои успехи — общее количество пройденных викторин, количество правильных ответов и общий процент точности.
- 💾 **Сохранение истории:** Все пользователи, сессии викторин и детальные ответы на вопросы сохраняются в базе данных PostgreSQL.
- 🐋 **Готов к развертыванию:** Полная поддержка Docker и Docker Compose для быстрого запуска вместе с базой данных и автоматическим применением миграций Alembic.

## 🛠 Технологический стек

- **Язык программирования:** [Python 3.11+](https://www.python.org/)
- **Фреймворк для Telegram-ботов:** [aiogram v3](https://docs.aiogram.dev/) (асинхронный)
- **База данных:** [PostgreSQL](https://www.postgresql.org/)
- **ORM:** [SQLAlchemy 2.0](https://www.sqlalchemy.org/) (асинхронный режим)
- **Миграции БД:** [Alembic](https://alembic.sqlalchemy.org/)
- **Интеграция с LLM:** [OpenAI Python SDK](https://github.com/openai/openai-python) (совместимый клиент для работы с API Gonkagate/OpenAI)
- **Валидация настроек:** [Pydantic Settings v2](https://docs.pydantic.dev/)
- **Контейнеризация:** [Docker](https://www.docker.com/) & [Docker Compose](https://docs.docker.com/compose/)

## ⚙️ Настройка окружения

Для работы приложения необходимы переменные окружения. Создайте файл `.env` в корневой директории на основе `.env.example`:

```bash
cp .env.example .env
```

Заполните переменные окружения в `.env`:

| Переменная | Описание | Пример значения |
|---|---|---|
| `BOT_TOKEN` | API Токен вашего Telegram-бота (полученный у @BotFather) | `123456789:ABCdefGhI...` |
| `DB_HOST` | Хост для подключения к PostgreSQL (для Docker укажите `db`) | `localhost` / `db` |
| `DB_PORT` | Порт PostgreSQL | `5432` |
| `DB_USER` | Имя пользователя базы данных | `postgres` |
| `DB_PASSWORD` | Пароль пользователя базы данных | `postgres` |
| `DB_NAME` | Название базы данных | `quizzard_db` |
| `GONKAGATE_BASE_URL`| Базовый URL API для интеграции с LLM | `https://api.gonkagate.com/v1` |
| `GONKAGATE_API_KEY` | Токен/ключ авторизации в API LLM | `gp-your-key-here` |
| `LLM_MODEL` | Название используемой языковой модели | `qwen/qwen3-235b-a22b-instruct-2507-fp8` |
| `LOG_LEVEL` | Уровень логирования проекта (`DEBUG`, `INFO`, `WARNING`, `ERROR`) | `INFO` |
| `ADMIN_CHAT_ID` | (Опционально) Telegram ID админа для получения логов от бота-логгера | `123456789` |
| `LOGS_BOT_TOKEN` | (Опционально) Токен отдельного Telegram-бота для пересылки всех логов админу | `987654321:XYZ...` |
| `LOG_FILE` | (Опционально) Имя файла для сохранения логов уровня WARNING и выше | `bot_logs.txt` |

## 🚀 Запуск проекта

Вы можете запустить проект двумя способами: локально с использованием виртуального окружения или через Docker.

### Способ 1: С помощью Docker Compose (Рекомендуемый)

Убедитесь, что у вас установлены [Docker](https://docs.docker.com/engine/install/) и [Docker Compose](https://docs.docker.com/compose/install/).

1. Заполните `.env` файл (укажите `DB_HOST=db`).
2. Запустите сборку и старт контейнеров:
   ```bash
   docker compose up -d --build
   ```
3. Docker автоматически:
   - Поднимет контейнер базы данных PostgreSQL и дождется его готовности.
   - Запустит миграции Alembic для создания необходимых таблиц.
   - Запустит Telegram-бота.

Посмотреть логи:
```bash
docker compose logs -f bot
```

Остановить проект:
```bash
docker compose down
```

---

### Способ 2: Локальный запуск (в виртуальном окружении)

Для этого способа у вас должна быть установлена и запущена база данных PostgreSQL.

1. Создайте и активируйте виртуальное окружение:
   ```bash
   python -m venv venv
   # Для Windows:
   venv\Scripts\activate
   # Для Linux/macOS:
   source venv/bin/activate
   ```
2. Установите зависимости:
   ```bash
   pip install --no-cache-dir -r requirements.txt
   ```
3. Примените миграции базы данных Alembic:
   ```bash
   alembic upgrade head
   ```
4. Запустите Telegram-бота:
   ```bash
   python -m src.main
   ```

## 🛠 Работа с базой данных и Alembic

Если вы изменяете модели данных в [`src/database/models.py`](file:///D:/projects/Quizzard/src/database/models.py), вам потребуется создать и применить новые миграции:

1. **Создание новой миграции:**
   ```bash
   alembic revision --autogenerate -m "describe_your_changes"
   ```
2. **Применение миграции к базе данных:**
   ```bash
   alembic upgrade head
   ```
3. **Откат последней миграции:**
   ```bash
   alembic downgrade -1
   ```

## 📖 Как играть

1. Откройте диалог с вашим ботом в Telegram и введите команду `/start`.
2. Нажмите кнопку **🎯 Начать викторину**.
3. Выберите уровень сложности: `🟢 Легкая`, `🟡 Средняя` или `🔴 Сложная`.
4. Введите тему викторины (например: *«Космос и астрономия»*).
5. Наблюдайте за процессом генерации в реальном времени — бот выведет список создаваемых вопросов.
6. Отвечайте на вопросы, выбирая один из четырех вариантов с помощью кнопок-букв (**A, B, C, D**).
7. После каждого ответа бот покажет результаты: зеленый маркер `🟢` для правильного ответа, красный `🔴` для вашей ошибки и белые `⚪️` для остальных. Также вы увидите подробное объяснение правильного решения.
8. Вы можете досрочно завершить игру с помощью кнопки **❌ Прервать викторину**.
9. В конце викторины вы увидите сводные результаты и сможете посмотреть свою общую статистику через команду `/stats`.
